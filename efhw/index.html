How this is organized

这套体系想做的事

关于 EFHW 的资料很多，但大多停留在两端：要么是厂商的宣传话术，要么是论坛里互相矛盾的经验之谈。一个动手的爱好者常常很难判断，某个说法到底是经过测量证实的、是从物理推导出来的，还是某个特定环境下的个例观察。

这份资料尝试把一条完整的认知链条走通：理论 → 动手 → 用数据观测自己的天线 → 对照理论 → 找到可改进处 → 复测验证。它建立在两块基础之上——一块是公开的 EFHW 理论文献（AA5TB、PA9X、N6CC、ON6URE 等前辈的工作），另一块是 PSK Reporter 上数十万条真实的传播报告。

我们对每一条结论都尽量标注它的证据来源与强度。无线电传播受太阳活动、季节、昼夜、对端设备等大量变量影响，很多观测只能算"在现有数据下的阶段性认识"，而非定论。下面这个标注体系会贯穿全站：

实测支持 有本站或第三方测量数据支撑 理论推导 由物理或几何计算得出，尚待实测 待验证 阶段性观察，样本或前提仍有限
一个前提：本站的实测部分以业余电台 BG1SB（北京 ON80）的一副 20m 端馈天线为主要研究对象。这意味着很多具体数字是"这一副天线、这个环境"的结果。方法可以借鉴，结论请勿直接套用到你自己的天线上——更好的做法是用同样的方法观测你自己的天线。

The Core Loop

贯穿全站的一条逻辑：可检验的闭环

这套体系最想传达的，不是某一个结论，而是一种做法——让关于天线的说法变得"可检验"。

  理论/几何推导            数据观测               对照与诊断
  ┌──────────────┐      ┌──────────────┐      ┌──────────────┐
  │ 由安装几何     │      │ PSK Reporter  │      │ 与同网格邻居   │
  │ 手算出方向图   │ ───▶ │ 真实报文作为   │ ───▶ │ 作差，看差异   │
  │ 主瓣/零点预测  │      │ 远场探针       │      │ 落在何处       │
  └──────────────┘      └──────────────┘      └──────┬───────┘
        ▲                                            │
        │                                            ▼
        │              ┌──────────────┐      ┌──────────────┐
        └───────────── │ 复测：改造后   │ ◀─── │ 找到可改进处   │
           闭环          │ 重跑同一分析   │      │ （朝向/架高等）│
                       └──────────────┘      └──────────────┘

这条闭环里有个关键设计：与同网格邻居作差。同一个网格里的其他电台，面对的是几乎相同的电离层、季节和对端人口分布。把自己的信号报告与他们作差，能在相当程度上抵消这些共同变量，让剩下的差异更多地反映天线、馈线与本地噪声的不同。这不是完美的隔离，但它把"我感觉信号变好了"这种主观判断，向"相对邻居的差值变化了多少 dB"这种可复现的度量推进了一步。

需要诚实说明的是：PSK Reporter 数据本身带有偏差——接收站的地理分布并不均匀，某些方向的报文多，未必是天线在该方向强，也可能只是那个方向的接收站和发射活动更密集。数据方法论一页会专门讨论这些局限，以及我们如何尽量减轻（而非消除）它们的影响。
